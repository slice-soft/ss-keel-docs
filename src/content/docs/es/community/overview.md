Keel es un proyecto open source mantenido por [SliceSoft](https://github.com/slice-soft). El repositorio comunitario ([slice-soft/ss-community](https://github.com/slice-soft/ss-community)) es la fuente de verdad para gobernanza, estándares y políticas compartidas entre repositorios.

## Ecosistema de repositorios

| Repositorio | Propósito |
|---|---|
| [ss-keel-core](https://github.com/slice-soft/ss-keel-core) | Librería core del framework |
| [keel](https://github.com/slice-soft/keel) | CLI de scaffolding |
| [ss-keel-docs](https://github.com/slice-soft/ss-keel-docs) | Sitio de documentación |
| addons `ss-keel-addon-*` | Módulos addon oficiales (cada uno en su propio repo) |

## Documentos de comunidad

### [GOVERNANCE.md](https://github.com/slice-soft/ss-community/blob/main/GOVERNANCE.md)

Describe la estructura del proyecto, roles de maintainers y cómo se toman decisiones, incluyendo el proceso de propuestas para cambios significativos y breaking changes.

### [CODE_OF_CONDUCT.md](https://github.com/slice-soft/ss-community/blob/main/CODE_OF_CONDUCT.md)

Estándares de comportamiento esperados para toda participación en el proyecto (issues, pull requests y demás espacios comunitarios), basado en Contributor Covenant 2.1.

### [VERSIONING.md](https://github.com/slice-soft/ss-community/blob/main/VERSIONING.md)

Cómo Keel aplica Semantic Versioning: qué se considera breaking change, cadencia de releases y relación entre versionado del core y addons.

### [SECURITY.md](https://github.com/slice-soft/ss-community/blob/main/SECURITY.md)

Proceso para reportar vulnerabilidades de forma responsable. Incluye versiones soportadas, divulgación coordinada y canales de contacto.

### [MAINTAINERS.md](https://github.com/slice-soft/ss-community/blob/main/MAINTAINERS.md)

Listado de maintainers activos, áreas de enfoque y camino para llegar a ser maintainer.

## Contribuir

Empieza leyendo [GOVERNANCE.md](https://github.com/slice-soft/ss-community/blob/main/GOVERNANCE.md) para entender proceso de propuesta y revisión. Luego abre un issue en el repositorio correspondiente antes de iniciar trabajo significativo.

Para cambios de contenido en este sitio, edita el `.md` relevante en `src/content/docs/` dentro de [ss-keel-docs](https://github.com/slice-soft/ss-keel-docs) y abre un pull request.
