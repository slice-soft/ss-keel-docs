Keel is an open source project maintained by [SliceSoft](https://github.com/slice-soft). The community repository ([slice-soft/ss-community](https://github.com/slice-soft/ss-community)) is the source of truth for governance, standards, and policies shared across repositories.

## Repository ecosystem

| Repository | Purpose |
|---|---|
| [ss-keel-core](https://github.com/slice-soft/ss-keel-core) | Core framework library |
| [keel](https://github.com/slice-soft/keel) | Scaffolding CLI |
| [ss-keel-docs](https://github.com/slice-soft/ss-keel-docs) | Documentation site |
| `ss-keel-addon-*` addons | Official addon modules (each in its own repo) |

## Community documents

### [GOVERNANCE.md](https://github.com/slice-soft/ss-community/blob/main/GOVERNANCE.md)

Describes the project structure, maintainer roles, and how decisions are made, including the proposal process for significant changes and breaking changes.

### [CODE_OF_CONDUCT.md](https://github.com/slice-soft/ss-community/blob/main/CODE_OF_CONDUCT.md)

Expected behavior standards for all participation in the project (issues, pull requests, and other community spaces), based on Contributor Covenant 2.1.

### [VERSIONING.md](https://github.com/slice-soft/ss-community/blob/main/VERSIONING.md)

How Keel applies Semantic Versioning: what is considered a breaking change, release cadence, and the relationship between core and addon versioning.

### [SECURITY.md](https://github.com/slice-soft/ss-community/blob/main/SECURITY.md)

Process for responsibly reporting vulnerabilities. Includes supported versions, coordinated disclosure, and contact channels.

### [MAINTAINERS.md](https://github.com/slice-soft/ss-community/blob/main/MAINTAINERS.md)

List of active maintainers, areas of focus, and the path to becoming a maintainer.

## Contributing

Start by reading [GOVERNANCE.md](https://github.com/slice-soft/ss-community/blob/main/GOVERNANCE.md) to understand the proposal and review process. Then open an issue in the relevant repository before starting significant work.

For content changes to this site, edit the relevant `.md` in `src/content/docs/` inside [ss-keel-docs](https://github.com/slice-soft/ss-keel-docs) and open a pull request.
