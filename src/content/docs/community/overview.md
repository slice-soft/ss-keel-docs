Keel is an open-source project maintained by [SliceSoft](https://github.com/slice-soft). The community repository ([slice-soft/ss-keel-community](https://github.com/slice-soft/ss-keel-community)) is the single source of truth for governance, standards, and cross-repo policies.

## Repository ecosystem

| Repository | Purpose |
|---|---|
| [ss-keel-core](https://github.com/slice-soft/ss-keel-core) | Core framework library |
| [ss-keel-cli](https://github.com/slice-soft/ss-keel-cli) | CLI scaffolding tool |
| [ss-keel-docs](https://github.com/slice-soft/ss-keel-docs) | This documentation site |
| [ss-keel-community](https://github.com/slice-soft/ss-keel-community) | Governance and community standards |
| `ss-keel-addon-*` addons | Official addon modules (each in its own repo) |

## Community documents

### [GOVERNANCE.md](https://github.com/slice-soft/ss-keel-community/blob/main/GOVERNANCE.md)

Describes the project structure, maintainer roles, and how decisions are made — including the proposal process for significant changes and breaking changes.

### [CODE_OF_CONDUCT.md](https://github.com/slice-soft/ss-keel-community/blob/main/CODE_OF_CONDUCT.md)

The standards we expect from everyone participating in the project — in issues, pull requests, and any other community venue. Based on the Contributor Covenant 2.1.

### [VERSIONING.md](https://github.com/slice-soft/ss-keel-community/blob/main/VERSIONING.md)

How Keel uses Semantic Versioning: what counts as a breaking change, the release cadence, and how addon versioning relates to the core.

### [SECURITY.md](https://github.com/slice-soft/ss-keel-community/blob/main/SECURITY.md)

How to responsibly disclose security vulnerabilities. Includes supported versions, the coordinated disclosure process, and contact details.

### [MAINTAINERS.md](https://github.com/slice-soft/ss-keel-community/blob/main/MAINTAINERS.md)

The list of active maintainers, their focus areas, and how to become a maintainer.

## Contributing

Start by reading [GOVERNANCE.md](https://github.com/slice-soft/ss-keel-community/blob/main/GOVERNANCE.md) to understand the proposal and review process. Then open an issue in the relevant repository before starting significant work.

For content changes to this documentation site, edit the relevant `.md` file under `src/content/docs/` in [ss-keel-docs](https://github.com/slice-soft/ss-keel-docs) and open a pull request.
